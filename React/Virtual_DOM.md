# 가상돔 (Virtual DOM)

- React의 고성능 렌더링을 위한 핵심 원리
- 실제 DOM의 구조를 모방한 메모리상의 가벼운 JavaScript 객체

## 동작 원리 (재조정, Reconciliation)
1.  상태 변경: 컴포넌트의 `state` 변경 발생
2.  가상돔 생성: 변경된 `state`를 기반으로 새로운 가상돔 트리 생성
3.  비교 (Diffing): 이전 가상돔과 새로운 가상돔을 비교하여 차이점 탐색
4.  일괄 업데이트: 찾아낸 차이점만 실제 DOM에 한 번에 적용 (조작 최소화)

## 장점
- 성능: 비용이 큰 실제 DOM 조작을 최소화하여 렌더링 성능 향상
- 개발 편의성: 개발자는 DOM 성능 최적화를 신경 쓸 필요 없이 UI 선언에만 집중

## 개념 예제
- 초기 상태: `state = { value: "A" }`
- 가상돔: `{ type: 'h1', children: 'A' }`
- 실제돔: `<h1>A</h1>`
<br/>
- 상태 변경: `state.value = "B"`
- 가상돔 비교: `{ children: 'A' }` -> `{ children: 'B' }` 변경 감지
- 실제돔 업데이트: `<h1>` 태그의 텍스트 내용만 'B'로 교체
